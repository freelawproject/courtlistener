Integration Proposal: U.S. Code Citation Hyperlinking & Link Service

This proposal outlines the integration of a local/hosted U.S. Code Link Service into CourtListener to hyper-link statutory references in case opinions, display them in an interactive side-drawer panel, include them in the Table of Authorities, and render quick-answer rich cards for search results.

1. Architectural Overview

The integration connects CourtListener's django server and front-end interface with a dedicated, lightweight U.S. Code Link Service API. This service ingests official XML schemas from the Office of the Law Revision Counsel (OLRC) and granular XML/HTML structures from GPO GovInfo, resolving and serving clean HTML segments of U.S. Code sections.

2. Key Features

1. Inline Hyperlinking in Opinions:

  • Extracts U.S. Code references using eyecite in annotate_citations.py.
  • Unmatched citations corresponding to U.S.C. are wrapped in a <span class="citation us-code-citation"> with data-title and data-section attributes.
  • Fallback hyperlink redirects to the U.S. Code Service's public HTML view for the section.

2. Interactive Side-Drawer Panel (Overlay):

  • Clicking a U.S. Code hyperlink intercepts the navigation and fetches the section text asynchronously via AJAX from the U.S. Code Service.
  • Content is displayed in a sliding drawer overlay panel without taking the user away from the opinion they are reading.

3. Statutory Table of Authorities:

  • Scans the pre-annotated opinion HTML to extract cited U.S. Code sections.
  • Renders a dedicated Statutory Authorities (U.S. Code) list under the Table of Authorities tab, sorted numerically by title and section.

4. Search Rich Cards:

  • Detects U.S. Code queries in searches (e.g., searching for 17 U.S.C. 101).
  • Fetches the corresponding section text and displays a Statutory Authority Quick Answer card above case search results.

3. Changes Introduced to CourtListener

The integration touches a few key components in the CourtListener codebase:

Component / File Purpose
cl/citations/annotate_citations.py Identifies unmatched FullLawCitation records with reporter U.S.C. and wraps them in custom span markup.
cl/settings/django.py Adds USCODE_API_URL environment setting to route server-side requests.
cl/settings/project/security.py Whitelists localhost/wildcard port 8000 in development mode inside the Content Security Policy connect-src list.
cl/assets/static-global/js/opinions.js Intercepts citation clicks, calls the AJAX API endpoint dynamically based on the current hostname, and opens the side drawer.
cl/opinion_page/templates/opinions.html Incorporates the HTML structure of the dismissible side-drawer panel overlay.
cl/assets/static-global/css/opinions.css Adds styles for the side-drawer panel, scroll container, and formats USLM/XML tag indentations (subsection, paragraph, etc.).
cl/opinion_page/views.py Parses opinion HTML for U.S. Code spans to populate the Table of Authorities context.
cl/opinion_page/templates/includes/opinion_tabs_content.html Renders the statutory list in the Table of Authorities tab.
cl/search/views.py Resolves search query citations to U.S. Code and requests section previews.
cl/search/templates/search.html Renders the rich quick-answer card for search results.

4. Current Scope & Limitations

IMPORTANT

Title 17 (Copyrights) Only During this initial phase, the U.S. Code Link Service is restricted to indexing Title 17 (Copyrights) for the year 2023. This limitation was introduced to keep the local test database lightweight and minimize storage/bandwidth requirements during development and evaluation. Future Scalability The parser and celery ingestion tasks in the service are fully generic. Adding support for all titles in the U.S. Code requires no code modifications—only running the ingestion command with the corresponding title numbers.

5. Local Evaluation & Testing

You can spin up a fully functioning local test environment containing the integration:

1. Start the environment:

   docker compose up -d

2. Seed the database (creates a test SCOTUS docket 26-9999 with Title 17 references):

   docker exec -i cl-django python manage.py shell < create_test_data.py

3. Verify Interactive Drawer:

  • Navigate to the test opinion at http://localhost:8080/opinion/3/acme-corp-v-beta-inc/.
  • Click on 17 U.S.C. § 101 and verify the side drawer opens with the statute text.

4. Verify Table of Authorities:

  • Go to http://localhost:8080/opinion/3/acme-corp-v-beta-inc/authorities/.
  • Verify the Statutory Authorities (U.S. Code) list is rendered and sorted.
