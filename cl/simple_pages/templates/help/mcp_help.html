Model Context Protocol (MCP) Server

CourtListener's MCP server gives AI assistants direct access to our legal databases—millions of court opinions, federal dockets, judge profiles, and more. Instead of relying on training data alone, your AI assistant can search and retrieve live data from CourtListener while you chat.

What is the Model Context Protocol?

The Model Context Protocol (MCP) is an open standard that lets AI applications connect to external data sources. When your AI assistant has CourtListener's MCP server installed, it can query our databases in real time to answer your legal research questions with current, verifiable information.

Compatible AI applications include Claude Desktop, Claude Code, Cursor, Windsurf, and any other application that supports MCP.

What Can It Do?

Search Across CourtListener

Search millions of records using the same powerful engine behind the CourtListener website:

  • Case Law—Search court opinions by keyword, citation, date range, court, judge, and more across hundreds of jurisdictions
  • RECAP Dockets & Documents—Search federal court dockets and filings from our RECAP Archive, containing millions of PACER documents
  • Judges—Find judges by name, court, appointer, political affiliation, law school, and other biographical details
  • Oral Arguments—Search oral argument recordings from appellate courts

Look Up Detailed Records

Retrieve complete information about any item in our database:

  • Full text of court opinions with metadata and citation information
  • Complete docket sheets including all entries, parties, and attorneys
  • Judge profiles with positions, education, and authored opinions
  • Court details including jurisdiction and coverage information
  • Financial disclosures, ABA ratings, and more

The MCP server provides access to over 20 API endpoints, giving your AI assistant the same data available to developers using our REST API.

Analyze & Verify Citations

A standout feature: the MCP server can extract and verify legal citations directly from any text.

  • Extract Citations—Parses legal text locally using eyecite to identify full case citations, short cites, id., supra, and statutory references. Runs entirely on your machine with no API calls.
  • Verify Citations—Cross-references extracted citations against CourtListener's database to confirm case names, dates, and validity. Ideal for checking AI-generated legal text or reviewing briefs.

Manage Alerts

Create and manage CourtListener alerts without leaving your AI assistant:

  • Search Alerts—Set up real-time, daily, weekly, or monthly alerts for case law and RECAP searches
  • Docket Alerts—Subscribe to updates on specific federal court dockets

Installation

The CourtListener MCP server runs locally on your computer and communicates with your AI application via standard input/output (stdio). You will need two things before you start:

  1. uv—A fast Python package manager. If you do not have it, install it by following the instructions at docs.astral.sh/uv.
  2. A CourtListener API token—Create a free account if you do not have one, then get your token from your API settings page.

Claude Desktop

  1. Open Claude Desktop and navigate to Settings > Developer > Edit Config. This will open your configuration file.
  2. Add the following to the file:
{
  "mcpServers": {
    "courtlistener": {
      "command": "uvx",
      "args": ["--from", "courtlistener-api-client[mcp]", "courtlistener-mcp"],
      "env": {
        "COURTLISTENER_API_TOKEN": "your-api-token"
      }
    }
  }
}
  1. Replace your-api-token with your actual token and save the file. Restart Claude Desktop for the changes to take effect.

Claude Code

Run this single command in your terminal:

claude mcp add courtlistener -e COURTLISTENER_API_TOKEN=your-api-token -- uvx --from "courtlistener-api-client[mcp]" courtlistener-mcp

Replace your-api-token with your actual token.

Cursor, Windsurf, and Other MCP Clients

Most MCP-compatible applications use a similar JSON configuration. Add a server entry with:

  • Command: uvx
  • Args: ["--from", "courtlistener-api-client[mcp]", "courtlistener-mcp"]
  • Environment variable: COURTLISTENER_API_TOKEN set to your token

Consult your application's documentation for where to place the MCP server configuration.

Authentication

The MCP server uses your CourtListener API token for authentication:

  • You need a free CourtListener account to use the MCP server.
  • Your existing API token works with the MCP server.
  • Standard API rate limits and usage policies apply to MCP requests just as they do to direct API calls.
  • Organizations with commercial API agreements can use MCP under the same terms.

If you do not yet have an account, you can create one for free. API tokens are available in your profile settings after registration.
